## 2.4 Sets

## 🌀 What Is a Set?

A **set** is an **unordered collection of unique elements**. No duplicates. No specific order. Its elements must be immutable (hashable).

The philosophical essence:

> **A set answers one question and one question only: Does this belong, or does it not?**

It does not care *where* something is. It does not care *how many times* it appears. It does not care about sequence. It only cares about **existence and membership**.

---

## 🧠 The Deep "Why"

Python introduced sets because there are problems where order is irrelevant, repetition is noise, and identity is the only thing that matters. Members of a tribe. Fundamental ideas. Ethical values. Permissions. In all of these, asking "in what position?" makes no sense. **The correct question is "does it belong?"**

---

## 🧾 Syntax

```python
my_set = {element1, element2, element3}

# Empty set requires set() — because {} creates an empty dictionary
empty_set = set()

# Converting from other collections (duplicates eliminated)
from_list = set([1, 2, 2, 3, 3, 3])  # {1, 2, 3}
```

### Fundamental characteristics:

**No duplicates** — each element appears only once. **No order** — you cannot access by index (`my_set[0]` raises an error). **Immutable elements** — only hashable objects (numbers, strings, tuples — NOT lists or dictionaries). **Mutable as a collection** — you can add and remove elements from the set itself.

Why no order? Technically, sets use hash tables internally, achieving O(1) lookups. Hash tables do not maintain order — they prioritize **search speed** over sequence. Philosophically: when you only care about "is it there or not?", order is unnecessary information. Eliminating it allows extreme optimization.

Why only immutable elements? Because sets need to compute hashes to store elements efficiently. Mutable objects could change after insertion, invalidating their hash and breaking the entire structure.

## 🛠️ Set Methods

**`add(element)`** — adds a single element. If it already exists, nothing happens. No error, no duplicate. Conscious membership.

**`update(iterable)`** — adds multiple elements from another collection. Fusion without manual looping.

**`remove(element)`** — removes an element. If it does not exist, raises an error. **Strict, conscious elimination** — you must be certain the element is there.

**`discard(element)`** — removes an element. If it does not exist, does nothing. **Tolerant elimination** — you do not care whether it is there or not. The crucial difference: `remove()` is strict and fails; `discard()` is flexible and never fails.

**`pop()`** — removes and returns an arbitrary element. Since sets have no order, there is no "last" element. Useful for processing elements when sequence does not matter.

**`clear()`** — empties the set. Reset without creating a new object.

## 🔍 Set Theory Operations

This is where sets truly shine. They implement the mathematical operations from set theory:

```python
a = {1, 2, 3}
b = {3, 4, 5}

# Union: all elements from both
a | b                    # {1, 2, 3, 4, 5}
a.union(b)

# Intersection: only elements common to both
a & b                    # {3}
a.intersection(b)

# Difference: elements in A but not in B
a - b                    # {1, 2}
a.difference(b)

# Symmetric difference: in one or the other, but not both
a ^ b                    # {1, 2, 4, 5}
a.symmetric_difference(b)
```

### Relationship methods:

**`issubset(other)`** — are all elements of A contained in B?
**`issuperset(other)`** — does A contain all elements of B?
**`isdisjoint(other)`** — do they share no elements at all?

## 🗝️ Accessing a Set

Sets do **not** support indexing or slicing. No `my_set[0]`. No `my_set[1:3]`.

The valid ways: **iteration** (traverse all elements in arbitrary order), **membership testing** with `in` (the most natural operation for sets), and **global functions** like `len()`, `max()`, `min()`, `sum()`.

```python
permissions = {"read", "write", "execute"}

# The most natural set operation
if "read" in permissions:
    print("Has read permission")

# Iteration (order may vary)
for permission in permissions:
    print(permission)
```

## 🧘 The Buddhist Metaphor

**A set is like the Sangha.** It does not matter who arrived first (no order). It does not matter how many times you come (no duplicates). It only matters **whether you are a member** (belonging). Everyone is equal in essence. The set treats each element as unique and irreplaceable.

### When to use sets:

Eliminating duplicates from a list. Fast membership checks (O(1)). Mathematical operations between groups (union, intersection). Representing membership (users, permissions, tags). Finding unique or common elements.

### When NOT to use sets:

You need to maintain order. You need to access by index. You need to count duplicates. You need mutable elements inside.

---

## ⚡ Essential Truths

> **Sets exist to answer: "Does this belong?"** Their lack of order is their optimization. Their rejection of duplicates is their discipline.

A set is not a list without order. It is a way of thinking in **pure identity**. It is set theory made code.
