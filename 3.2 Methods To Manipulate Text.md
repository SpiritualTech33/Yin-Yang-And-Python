# 🐍 The Yin Yang of Python

## 3.2 Methods for Manipulating Text

---

The guiding idea before we begin: text methods do not exist to "change strings." They exist to **change the relationship between text and its meaning**.

Python offers dozens of string methods. We will organize them into three categories by purpose: **Format** (how text looks), **Cleaning and Modification** (transforming content), and **Search and Validation** (finding and verifying).

---

## 🎨 Category 1: Format Methods

These methods do not change what is said — they change how it is presented.

**`upper()`** — converts all letters to uppercase. Exists for standardization — comparing inputs regardless of how the user typed them. **`lower()`** — converts all letters to lowercase. Exists for normalization — creating uniformity for searches. **`capitalize()`** — first letter uppercase, rest lowercase (only the first letter of the entire string, not each word). **`title()`** — first letter of every word capitalized. **`swapcase()`** — inverts case entirely. **`center()`, `ljust()`, `rjust()`** — alignment with padding characters.

```python
mantra = "om mani padme hum"
mantra.upper()       # "OM MANI PADME HUM"
mantra.lower()       # "om mani padme hum"
mantra.capitalize()  # "Om mani padme hum"
mantra.title()       # "Om Mani Padme Hum"

"Om".center(20, "-")  # "---------Om---------"
```

---

## 🧹 Category 2: Cleaning and Modification

Here the alchemy happens. These methods transform content — they remove, replace, divide.

**`strip()`** — removes spaces (and other characters) from both ends. Variants: `lstrip()` (left only), `rstrip()` (right only). Essential for cleaning user input.

**`replace(old, new)`** — finds a substring and replaces it with another. Can take an optional limit for number of replacements.

**`split(separator)`** — breaks the string into a list based on a separator (default: space). **`join(list)`** — the inverse of `split()` — unites list elements with a separator. These two form a beautiful Yin-Yang pair: `split()` (Yang) divides the whole into parts; `join()` (Yin) fuses parts into a whole.

**`splitlines()`** — splits multiline text into a list of lines.

```python
"   Yamil   ".strip()                    # "Yamil"
"The error is your enemy".replace("enemy", "teacher")
                                         # "The error is your teacher"

"Om Mani Padme Hum".split()             # ['Om', 'Mani', 'Padme', 'Hum']
" | ".join(["Buddhism", "Taoism", "Zen"])# "Buddhism | Taoism | Zen"

"Buddhism,Taoism,Stoicism".split(",")    # ['Buddhism', 'Taoism', 'Stoicism']
```

---

## 🔍 Category 3: Search and Validation

These methods do not transform — they ask. They find and verify, answering questions about the text.

**`find(substring)`** — returns the index where a substring starts; returns `-1` if not found. **`index(substring)`** — same as `find()`, but raises an error if not found. Use `find()` when it might not exist; use `index()` when it *must* exist. **`count(substring)`** — counts how many times a substring appears.

**`startswith(prefix)`** — checks if the string begins with something. **`endswith(suffix)`** — checks if it ends with something. Both accept tuples for multiple options.

**Validation methods:** `isalpha()` (only letters?), `isdigit()` (only digits?), `isalnum()` (only letters and numbers?), `isspace()` (only whitespace?).

**`in` operator** — checks if a substring exists within the string. The most natural search.

```python
text = "The Tao of Python"
text.find("Tao")          # 4
text.count("o")           # 2
text.startswith("The")    # True
text.endswith(".py")      # False
"Tao" in text             # True

"Python3".isalpha()       # False
"24".isdigit()            # True
```

---

## 🧘 The Final Metaphor

String methods are like the tools of a Zen gardener. **Format** methods give aesthetic form — pruning, aligning. **Cleaning** methods remove what is unnecessary — clearing dead leaves. **Search** methods observe with attention — finding the hidden flower.

Manipulating text is not mastering methods. It is learning to **listen better** to what arrives, **express better** what leaves, and **respect form** without losing meaning. Python does not ask you to force language. It asks you to dialogue with it.
