## 1.4 Variables and Data Types

---

## 🧠 What Are Variables?

A **variable** is a **name you give to a piece of memory** where you store information. It is a labeled box where you place something to use later.

```python
name = "Yamil"
age = 24
```

`name` and `age` are **variables** — labels that point to data in memory.

---

## 🤔 Why Do Variables Exist?

### Programming Without Variables (1940s–50s):

The earliest computers had no variables. You programmed directly with **memory addresses**:

```assembly
LOAD 1000    # Load the datum at memory position 1000
ADD 1004     # Add the datum at position 1004
STORE 1008   # Store the result at position 1008
```

The problems were brutal: **illegible** (what is at position 1000 — age? price? temperature?), **fragile** (if positions shift, all your code breaks), **unscalable** (large projects become impossible), and **error-prone** (type 1004 instead of 1040 and you have a ghost bug).

---

### The Deep "Why"

**1. Abstraction over hardware.** Without variables, you think "position 1000 contains 24" — machine thinking. With variables, you think "age is 24" — human thinking. **Variables are the bridge between how humans think and how machines function.**

**2. Reuse of information.** Without variables, you write "Yamil" in a hundred places. If it changes to "Yamil García," you change it in a hundred places. With variables, you change it once.

**3. Semantic clarity.**

```python
# Without variables (magic numbers):
if 24 > 18:
    print("Can vote")

# With variables (clear intention):
age = 24
legal_age = 18

if age > legal_age:
    print("Can vote")
```

The second version **explains itself**.

**4. Modifiability.** Change `discount = 0.30` in one place and the entire calculation adjusts automatically.

---

## 📊 What Are Data Types?

**Data types** are categories that tell Python **what kind of information** you are storing.

The basic types: **int** (integers: `24`, `-5`), **float** (decimals: `3.14`, `-0.5`), **str** (text: `"Yamil"`, `"Hello world"`), **bool** (booleans: `True`, `False`).

---

### Why Do Types Exist?

**1. Error prevention.** Types detect mistakes before they cause disasters. If `price = "100"` (a string, not a number), Python knows something is wrong when you try to do arithmetic with it.

**2. Memory optimization.** Different types occupy different space — Python optimizes internally according to type.

**3. Appropriate operations.** Each type has valid operations: numbers can be summed; strings can be concatenated and sliced; booleans support logical operations. Without types, we would not know what operations are valid.

**4. Clarity of intention.** `is_adult = True` is clearly a condition. `name = "Yamil"` is clearly text. `age = 24` is clearly a whole number. `average = 8.5` is clearly a decimal. The type communicates meaning.

---

## 🔑 The Deepest Truth

> **Variables and types are the way humans domesticated the computer.**
>
> Without them, programming would be like speaking in raw binary. With them, we express ideas in a language that both humans and machines understand.
>
> **They are the first level of abstraction that makes everything else in programming possible.**

## 📝 Rules for Writing Variables

**Mandatory (technical):** only letters, numbers, and `_`; cannot start with a number; cannot use reserved words; case-sensitive.

**PEP8 conventions (professional):** `snake_case` for variables, `UPPERCASE` for constants, descriptive names, avoid cryptic abbreviations, verbs for booleans, plurals for collections, appropriate length.

### 💎 The Golden Rule

> **If someone reads your code without context, do they understand what each variable does?**
> If yes, you named it well. If no, rename it.

## 💬 What Are Comments?

**Comments** are text Python ignores completely — **messages for humans**, not for the machine. Use `#` for inline comments and `"""docstrings"""` for functions and classes.

Comments exist because code is read **ten times more** than it is written, humans forget (even the author), non-obvious decisions need context, and collaboration requires others to understand your code.

**The philosophy:** explain WHY, not WHAT. Document non-obvious decisions. Keep comments updated. Do not comment on the obvious. Do not use comments to "fix" bad code.

> **The best comment is the one you do not need to write because the code explains itself.** But when you *do* need a comment, make it clear, concise, and useful.

## 🔄 What Is Type Conversion?

**Type conversion** is Python's ability to convert one data type into another. When you request a number from a user with `input()`, you receive a string — so you need conversion:

```python
user_input = int(input("Enter your number here:"))
```

### Why Do Conversions Exist?

**The outside world speaks in text.** User input, text files, web APIs, databases — everything that enters or exits a program arrives as strings. Conversions are the **bridge between the external world (text) and the internal world (specific types)**.

### Main Conversion Functions:

```python
int()     # To integer
float()   # To decimal
str()     # To text
bool()    # To boolean
list()    # To list
tuple()   # To tuple
set()     # To set
dict()    # To dictionary
```

Without type conversions, you could not process user input, perform calculations with text data, save numbers to files, or mix types in operations. They exist because Python is dynamically typed — you do not declare types, but sometimes you need to change them explicitly.
