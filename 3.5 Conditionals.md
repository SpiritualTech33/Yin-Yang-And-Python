## 3.5 Conditionals (if / elif / else)

## 🌓 The Birth of Choice

A plant does not choose; it grows toward light. An animal chooses — flee or fight, hunt or rest. And humans choose constantly.

A conditional is a **well-asked question**, in the Socratic sense. *If this is true… what follows? If it is not… what alternative exists? And if none… what do we do then?* `if`, `elif`, `else` are not just keywords. They are forms of reasoning.

---

## IF: The Fundamental Question

`if` evaluates a condition. If it is `True`, the indented block executes. If it is `False`, the block is skipped. Silence.

```python
temperature = 35

if temperature > 30:
    print("It's hot — hydrate well")

minutes_meditation = 28

if minutes_meditation >= 20:
    print("✅ Practice complete — amulet activated")
```

Compound conditions use `and` (both must be True), `or` (at least one), and `not` (inverts the value):

```python
hours_sleep = 7
energy = 8

if hours_sleep >= 7 and energy >= 7:
    print("🔥 Optimal day — train intensely")
```

---

## ELSE: The Alternative

`else` means "if not..." — the path taken when the `if` condition is `False`.

```python
age = 17

if age >= 18:
    print("✅ You can register")
else:
    print("❌ You must be of legal age")

consecutive_days = 12
goal = 15

if consecutive_days >= goal:
    print(f"🔱 Amulet unlocked! Streak: {consecutive_days} days")
else:
    remaining = goal - consecutive_days
    print(f"⏳ {remaining} days left to unlock the amulet")
```

---

## ELIF: The Multiple Path

`elif` (else if) means: "If not the above, then perhaps this?" For when you have more than two possibilities. Python evaluates in order — when it finds the first `True` condition, it executes that block and skips the rest.

```python
batting_avg = 0.350

if batting_avg >= 0.400:
    category = "Legendary 🏆"
elif batting_avg >= 0.350:
    category = "Elite ⭐"
elif batting_avg >= 0.300:
    category = "Excellent ✅"
elif batting_avg >= 0.250:
    category = "Average 📊"
else:
    category = "Needs improvement 💪"

print(f"Your average {batting_avg:.3f} is: {category}")
# Your average 0.350 is: Elite ⭐
```

**Golden rule:** place the most restrictive conditions first. If `age > 18` is checked before `age > 21`, the second will never execute.

---

## Nested Conditionals: Decisions Within Decisions

```python
has_amulet = True
streak_days = 20

if has_amulet:
    if streak_days >= 15:
        print("🔱 Amulet activated — full power")
    else:
        print("⏳ Amulet inactive — continue your streak")
else:
    print("❌ You do not have this amulet yet")
```

Like Matryoshka dolls — each decision can contain sub-decisions. But be careful: too many levels makes code hard to read. Maximum recommended: 2-3 levels of nesting.

---

## Boolean Expressions: Pure Logic

Conditionals evaluate expressions that return `True` or `False`. Comparison operators (`==`, `!=`, `>`, `<`, `>=`, `<=`) and logical operators (`and`, `or`, `not`) combine into complex evaluations:

```python
consecutive_days = 108
total_hours = 150
chakras_mastered = 7
has_teacher = True

if (
    consecutive_days >= 108
    and total_hours >= 100
    and (chakras_mastered == 7 or has_teacher)
):
    print("🏆 LEGENDARY AMULET UNLOCKED")
```
